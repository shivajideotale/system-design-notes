# Phase 2: Core System Design Concepts (Weeks 5–10)

This phase focuses on fundamental patterns and principles for designing scalable, available, and performant systems.

## Topics

- [Scalability Patterns](scalability-patterns.md)
- [Back-of-the-Envelope Estimation](back-of-the-envelope-estimation.md)
- [High Availability Patterns](high-availability-patterns.md)
- [CAP & PACELC Theorem](cap-pacelc-theorem.md)
- [Rate Limiting & Throttling](rate-limiting-throttling.md)

## Learning Objectives

By the end of this phase, you should be able to:
- Apply scalability patterns to grow systems horizontally and vertically
- Perform capacity planning with back-of-the-envelope calculations
- Design systems with high availability and fault tolerance
- Understand trade-offs in distributed systems (CAP/PACELC)
- Implement rate limiting for API protection and fairness
